# OpenSunstar v0.1.0 Release Notes

**Release date:** 2026-06-19  
**License:** MIT

---

## Overview

OpenSunstar v0.1.0 is the **first public release** — a native desktop control plane for AI coding CLI tools. Manage provider switching, Agent configuration (MCP / Skills / Prompts), and multi-repo portfolio insights in one app.

**Download:** [GitHub Releases](https://github.com/alisunstar/OpenSunstar/releases/tag/v0.1.0)

---

## Highlights

### Connect in minutes

- **Simple Connect** wizard: supplier → API key → apply to CLI
- **50+ provider presets** across seven supported tools
- **System tray** quick switch; Claude Code hot-switch without restart

### Unified Agent workspace

- MCP panel with discovery registry and per-app sync
- Skills from GitHub, ZIP, skills.sh, ClawHub, ModelScope
- Prompts, Commands, Hooks, Permissions, Subagents, Sessions

### Portfolio & operations

- **Portfolio dashboard** — multi Git repo cockpit (not a task kanban)
- Unified **7-day commit metrics** across cards, matrix, and AI weekly reports
- Usage dashboard, budget alerts, WebDAV / S3 cloud sync

### Built for reliability

- SQLite storage with atomic writes and auto-backup
- Local proxy, auto-failover, circuit breaker
- Windows · macOS · Linux (Tauri 2)

---

## Supported tools

Claude Code · Claude Desktop · Codex · Gemini CLI · OpenCode · OpenClaw · Hermes

---

## Upgrade note

This is the initial release. There is no migration from a prior OpenSunstar version. Existing CLI configs are imported as the **default** provider on first launch.

## Known limitations (v0.1.0)

- Most CLI tools require a **terminal restart** after provider switch (except Claude Code hot-switch).
- Portfolio stage / MVP progress is stored in localStorage; SQLite migration planned for a future release.
- Auto-sync error toasts for WebDAV/S3 background sync surface in settings UI, not as global notifications.
